# v0.4 Plan

v0.4 is an API polish release. The package is already published, so this phase
keeps changes additive and focused on ergonomics, cancellation hardening, and
clearer release documentation.

## Scope

- Add modern config helpers:
  - `Expr.min_entropy(threshold)`
  - `Expr.token_efficiency(...)`
  - `Expr.finding_contains_any(...)`
  - `Expr.finding_matches_any(...)`
  - `Expr.attribute_contains_any(...)`
  - `Expr.attribute_matches_any(...)`
  - `Expr.path_matches_any(...)`
  - `Expr.git_commit_in(...)`
  - `Expr.any_of(...)`, `Expr.all_of(...)`, and `Expr.not_(...)`
  - `Rule.regex_rule(...)`
  - `Rule.path_rule(...)`
  - `Rule.prefixed_token_rule(...)`
  - `Rule.pem_private_key_rule(...)`
- Add `Validation` helpers for common structured validation results and a
  bearer-token GET validator expression.
- Add `Rule.entropy` serialization for compatibility with Betterleaks'
  `entropy = ...` TOML field.
- Add `extend_base_path` handling so inline configs can serialize relative
  `[extend].path` values predictably.
- Harden async cancellation cleanup by shielding the executor future, requesting
  native cancellation by request id, and draining late native exceptions.
- Update roadmap, backlog, configuration docs, and release metadata for
  `0.4.0`.

## Non-Goals

- No streaming scan API in v0.4.
- No new Git scopes beyond `scope="worktree"`.
- No musllinux/Alpine wheels.
- No runtime subprocess-backed scan mode.
- No Betterleaks version bump unless explicitly planned and tested.

## Config Decisions

Prefer modern namespaced Expr filters for new configs. Betterleaks filters
return `True` when a finding should be skipped, so `Expr.min_entropy(3.5)`
serializes a skip-when-too-low predicate:

```python
Expr.min_entropy(3.5)
```

```text
filter.entropy(finding["secret"]) <= 3.5
```

`Rule.entropy` remains useful for users porting existing Betterleaks TOML, but
upstream Betterleaks translates legacy entropy fields into filters internally.
Legacy `allowlists` are not modeled as dataclasses. Use `Expr.path_matches_any`,
`Expr.git_commit_in`, `Expr.finding_matches_any`, and
`Expr.finding_contains_any` in `prefilter` or `filter` instead.

## Acceptance Criteria

- Public package version is `0.4.0`.
- `BetterleaksConfig.to_toml()` serializes the new helper-created rules and
  `entropy` field.
- Helper-generated configs are accepted by the native Betterleaks bridge.
- Invalid entropy thresholds fail in Python before crossing the native ABI.
- Empty helper inputs fail in Python with `ConfigFormatError`.
- Relative inline `extend.path` values can be resolved with `extend_base_path`
  and are resolved automatically when using `BetterleaksConfig.write(...)`.
- Async cancellation still calls `BetterleaksCancel(request_id)`.
- coverage-gated `pytest`, `ruff`, `mypy`, docs build, and wheel build pass.
- Backlog keeps streaming, additional Git scopes, provider scans, and
  musllinux out of the release scope, while config coverage for Betterleaks
  `v1.6.1` stable fields is marked done.
